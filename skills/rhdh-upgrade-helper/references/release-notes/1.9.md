# RHDH 1.9 Release Notes

Red Hat Developer Hub 1.9 is a fully supported, enterprise-grade productized version of upstream Backstage 1.45.3.

Source: <https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.9/html/red_hat_developer_hub_release_notes/index>

## New features and enhancements

- **Optional cache label filter**: Limits Operator cache to Secrets/ConfigMaps labeled `rhdh.redhat.com/external-config=true`. Reduces memory consumption.
- **Scaffolder Backend Module HTTP Request from RoadieHQ is now GA supported**
- **Community plugins migration to GitHub Container Registry**: Community-supported plugins moved from bundled wrappers to OCI images on ghcr.io.
- **Bulk Import plugin interface redesign**: Simplified repository import workflow.
- **Comprehensive guide for creating dynamic plugins for RHDH**
- **Floating "Lightspeed" button**: Globally accessible button for AI assistance from any page. Overlay, docked, or fullscreen display modes.
- **Developer Lightspeed chat history management**: Rename, delete, pin chats. MCP tool calling trail visualization.
- **GitLab authentication and user group ingestion**: GitLab as identity provider with auto-provisioned user/group data.
- **Simplified plugin management**: `!path` suffix no longer needed for single-plugin OCI packages. `{{inherit}}` tag support for OCI image references.
- **Support for mirroring RHDH dynamic plugin OCI artifacts**: `mirror-plugins.sh` script for disconnected environments with two-phase workflow (export/import).
- **Custom catalog transformer plugins supported**
- **Auto-logout mechanism for inactive users**: Configurable idle session timeout with pre-logout warning.
- **Aggregated scorecard metrics by ownership**: Save metrics to database with configurable collection frequency. Homepage KPI cards.
- **ServiceNow plugin supports configurable column mapping**
- **Bulk Import plugin supports Orchestrator workflows**
- **Japanese language support**
- **Argo CD plugin with multi-cluster support**: Discover applications across multiple instances/namespaces.
- **Template update pull requests**: Auto-create PRs when components are out of sync with source template.
- **Loki logs integration for Orchestrator workflows**
- **Italian language support**
- **Workflow deployment automation with Orchestrator**: Basic workflow bootstrap software template.
- **Secure service-to-service authentication**: Internal plugin-to-plugin and external service-to-plugin auth.
- **Backstage Operator supports StatefulSet deployments**: `spec.deployment.kind: StatefulSet` in v1alpha5 API.
- **PostgreSQL 17 support**: Validated on AWS RDS, Azure, GCP Cloud SQL.
- **Operator CR supports v1alpha5 API version**: Removes deprecated `spec.application.image`, `spec.application.replicas`, `spec.application.imagePullSecrets`.
- **Unauthenticated access to RHDH images**: Use `registry.access.redhat.com` instead of `registry.redhat.io`.

## Technology Preview features

- **OCI artifacts for dynamic plugins**: Switch from bundled plugin wrapper to OCI artifact. **Do not enable both simultaneously.**
- **GitHub Events Module for catalog**: `@backstage/plugin-events-backend-module-github` for event-driven catalog updates.

## Developer Preview features

- **OpenShift AI Connector annotation values updated**
- **Quay plugin multi-instance capability**
- **Catalog entity registration/unregistration MCP tools**
- **Software template for agentic AI application with Llama Stack**
- **Quay and Argo CD plugin updates**: Argo CD plugin renamed (removed `redhat` prefix from NPM package names).
- **RHDH must-gather tool**: Diagnostic data collection.
- **Support for multiple identity providers on login screen**
- **Dynamic plugin management in RHDH Local**

## Deprecated features

- **Helm chart template helpers renamed**: `janus-idp.*` → `rhdh.*` (e.g., `janus-idp.hostname` → `rhdh.hostname`). Old names still accepted but will be removed.
- **Backstage CR version v1alpha3 deprecated**
- **Global Floating Action Button plugin deprecated**: Use Global Header Quick links instead.
- **Community plugins migration to OCI images**: Community-supported plugins no longer bundled in container image. Must use `oci://ghcr.io/...` references. Tech Preview and GA plugins remain bundled. See Community plugins migration table.

## Removed features (breaking changes)

- **Operator users with Orchestrator 1.7/1.8 require manual dynamic-plugins ConfigMap update** after Operator upgrade to 1.9.
- **Helm upgrade may require manual update of custom values files**: `upstream.backstage.extraVolumeMounts`, `upstream.backstage.extraVolumes`, `upstream.backstage.initContainers` lists are not deep-merged by Helm. Must include new defaults for Extensions Catalog Index: `extensions-catalog` volume/mount, `CATALOG_INDEX_IMAGE` and `CATALOG_ENTITIES_EXTRACT_DIR` env vars.
- **Azure DevOps Scaffolder module replaces Azure Repositories module**: `@parfuemerie-douglas/scaffolder-backend-module-azure-repositories` incompatible with Backstage 1.45.
- **Bundled community-supported plugins removed**: Must use `oci://ghcr.io/...` paths.
- **Marketplace plugins renamed to Extensions**: `backstage-plugin-marketplace` → `backstage-plugin-extensions`.
- **CRD v1alpha1 and v1alpha2 versions removed**
- **Plugin support level changes**:
  - TP → Community: `roadiehq-scaffolder-backend-argocd`, `backstage-plugin-scaffolder-backend-module-azure`, `backstage-community-plugin-servicenow`
  - GA → Community: `backstage-community-plugin-quay`, `backstage-community-plugin-scaffolder-backend-module-quay-dynamic`, `backstage-community-plugin-tekton`, `roadiehq-backstage-plugin-argo-cd-backend`
- **Red Hat backend plugin for Argo CD replaces RoadieHQ plugin**

## Known issues

- **Orchestrator in-place upgrade fails on immutable database-creation job** (RHDHBUGS-3325): Delete the completed job and re-run upgrade. Fixed in 1.10.
- **Lightspeed plugin does not display Tool Call and Thinking sections** (RHDHBUGS-2914)
- **Operator-backed instances may fail to start with mirrored plugin catalog index image** (RHDHBUGS-2738): Workaround — empty the default plugins list in Operator's ConfigMap.
- **Switching deployment kind requires manual deletion of previous resource** (RHDHBUGS-2734)
- **RHDH with multiple replicas fails to start** (RHDHBUGS-2617): Race condition during DB migrations. Start with 1 replica, then scale up.
- **ArgoCD deployment fails due to PostgreSQL serviceBindings incompatibility** (RHDHBUGS-2438): Disable serviceBindings and create pre-existing secret.

## Fixed issues (upgrade-relevant)

- **Pulling images no longer requires auth secret** (RHDHBUGS-2767): Default registry changed to `registry.access.redhat.com`.
- **Catalog index signature errors fixed** (RHDHBUGS-2799)
- **DynamicArtifact paths and lifecycle statuses corrected** (RHDHBUGS-3037): Bundled plugins incorrectly marked as Community Supported now restored to GA/TP.
- **OAuth cookies configured as host-only** (RHDHBUGS-945): Fixes cross-subdomain session issues.
- **Plugins using alpha language API render correctly** (RHDHBUGS-2495)
