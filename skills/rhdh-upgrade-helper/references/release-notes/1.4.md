# RHDH 1.4 Release Notes

Red Hat Developer Hub 1.4 is a fully supported, enterprise-grade productized version of upstream Backstage 1.32.6.

Source: https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.4/html/release_notes/index

## New features and enhancements

- **Individual mountPath for extra ConfigMaps or Secrets**
- **PersistentVolumeClaims (PVC) support**
- **Removed kube-rbac-proxy sidecar**: Operator Pod resources reduced.
- **`developerHub.flavor` field**: Identify Backstage flavor (rhdh, rhtap, vanilla).
- **PVC management in Operator**: Mount directories from pre-created PVCs via `spec.application.extraFiles.pvcs`.
- **Red Hat Build of Keycloak authentication**: Keycloak supports ingesting users and groups.
- **Third-party plugin installation**: Install plugins without rebuilding the RHDH application.
- **Catalog backend module logs plugin enabled**: Converted to static plugin for better performance.
- **Google Kubernetes Engine (GKE) fully supported**: Out of Developer Preview.
- **Concurrent dynamic plugin installation**: Multi-replica RHDH with Persistent Volume for Dynamic Plugins cache now possible.

## Breaking changes

- **Metrics port changed from 7007 to 9464**: Prom-client replaced with OpenTelemetry metrics. Deprecated metrics removed. Update Prometheus queries.
- **Plugins with updated scope** (`@janus-idp` → `@backstage-community` and `@red-hat-developer-hub`):
  - `@janus-idp/backstage-plugin-acr` → `@backstage-community/plugin-acr`
  - `@janus-idp/backstage-plugin-rbac` → `@backstage-community/plugin-rbac`
  - `@janus-idp/backstage-plugin-tekton` → `@backstage-community/plugin-tekton`
  - `@janus-idp/backstage-plugin-topology` → `@backstage-community/plugin-topology`
  - `@janus-idp/backstage-plugin-quay` → `@backstage-community/plugin-quay`
  - `@janus-idp/backstage-plugin-keycloak-backend` → `@backstage-community/plugin-catalog-backend-module-keycloak`
  - `@janus-idp/backstage-plugin-bulk-import` → `@red-hat-developer-hub/backstage-plugin-bulk-import`
  - `@backstage/plugin-tech-radar` → `@backstage-community/plugin-tech-radar`
  - `@backstage/plugin-sonarqube` → `@backstage-community/plugin-sonarqube`
  - Plus many more — see full migration table in release notes.
  - Dynamic plugin configuration also changed accordingly.

## Deprecated features

- **`janus-idp-backstage-plugin-aap-backend-dynamic` deprecated**: Use Ansible plug-ins for RHDH instead.
- **Audit log rotation deprecated**: Evaluate platform log forwarding solutions.
- **Red Hat Single-Sign On 7.6 deprecated as auth provider**: Migrate to Red Hat Build of Keycloak v24.

## Technology Preview features

- **Notification backend plugins**: `@backstage/plugin-signals`, `@backstage/plugin-notifications-backend`, `@backstage/plugin-notifications`, `@backstage/plugin-notifications-backend-module-email`. Disabled by default.

## Known issues

- **Multi-Attached error for Volume (PVC)**: When using PVC for dynamic plugins cache.
- **Topology plugin permission not displayed in RBAC UI**

## Fixed issues (upgrade-relevant)

- **Operator channel upgrade workaround** (RHIDP-4687): Delete subscription, create new one pointing to new channel.
- **Table lock on Operator deployment fixed** (RHIDP-5411)
- **RBAC plugin compatibility with Backstage 1.31/1.32** (RHIDP-4241)
- **RBAC API consistency across multi-pod deployments** (RHIDP-4734): Redis cache and traffic routing.
- **Tech Radar plugin now requires both frontend and backend plugins** (RHIDP-5297)
