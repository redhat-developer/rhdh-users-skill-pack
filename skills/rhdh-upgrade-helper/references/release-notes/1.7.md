# RHDH 1.7 Release Notes

Red Hat Developer Hub 1.7 is a fully supported, enterprise-grade productized version of upstream Backstage 1.39.1.

Source: <https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.7/html/red_hat_developer_hub_release_notes/index>

## New features and enhancements

- **Improved plugin configuration merging for include files**: Configurations from include files are now merged (preserving existing keys) instead of overwritten.
- **Orchestrator 1.7 available**: As of RHDH 1.7.1, Orchestrator plugin 1.7.1 is available.
- **Full support for Adoption Insights plugins**: No longer Technology Preview — fully supported and enabled by default.
- **Dynamic plugin dependency management**: Operator can manage Kubernetes resources required by plugins.
- **Orchestrator plugins with dependencies added to RHDH profile**: Disabled by default; enabling deploys dependencies automatically.
- **Updated RHDH theme**: `@redhat-developer/red-hat-developer-hub-theme` replaced with plugin version from rhdh-plugins repository. Minor UI updates (border radius on cards).
- **Improved RBAC usability**: Aligned table heights, standardized icon colors, simplified page titles, improved Conditional Access sidebar, preserved search input.
- **Plugin management via Extensions UI**: Administrators can enable, disable, install, or edit dynamic plugin configuration. Opt-in, single-replica, PVC-based. Not for production.
- **Dynamic plugins GA + new CLI**: Backend dynamic plugins reach GA. `@red-hat-developer-hub/cli` (rhdh-cli) released. `@janus-idp/cli` deprecated.
- **Quick Start for platform engineers**: Guided interactive setup for administrators on first login. Tracks progress, context-aware help.
- **Improved global header UX**: User avatar in profile dropdown.
- **PatternFly 6 visual alignment**: Updated look and feel.
- **Versioning for Software Templates**: `backstage.io/template-version` annotation and `catalog:template:version` action.
- **HA support for Amazon EKS**: Scale beyond single replica.
- **Adoption Insights enabled by default**: Key metrics and trends for usage understanding.
- **Community plugins updated to Backstage 1.39**
- **Improved dynamic plugin installation robustness**: Warning and skip instead of failure when include file not found.
- **Full support for Microsoft Graph catalog plugin**: `@backstage/plugin-catalog-backend-module-msgraph` for user/group synchronization.
- **New Home page cards**: EntitySection, OnboardingSection, TemplateSection.

## Breaking changes

- **Optional Secret `dynamic-plugins-npmrc` for Operator removed**: Must now modify default config or mount Secret manually with `NPM_CONFIG_USERCONFIG` env var. See workaround in release notes.

## Deprecated features

- **Backstage CR versions v1alpha1 and v1alpha2**: Deprecated in 1.7, will be removed in 1.9.
- **Deprecation of Bundled Plugin Wrappers**: Transitioning to independent OCI artifacts. No breaking changes in 1.7 — all bundled plugins continue to work. Customers encouraged to prepare for removal in 1.8+.
- **`@janus-idp/cli` deprecated**: Replaced by `@red-hat-developer-hub/cli`.

## Technology Preview features

None in this release.

## Developer Preview features
