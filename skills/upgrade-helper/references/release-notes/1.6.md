# RHDH 1.6 Release Notes

Red Hat Developer Hub 1.6 is a fully supported, enterprise-grade productized version of upstream Backstage 1.36.1.

Source: https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.6/html/red_hat_developer_hub_release_notes/index

## New features and enhancements

- **OpenTelemetry metrics support for Keycloak backend plugin**: Counters for fetch task failures and batch failures with `taskInstanceId` label. Export via any OpenTelemetry-compatible backend (e.g., Prometheus).
- **Enhanced session duration control**: New `sessionDuration` configurable field for authentication providers. Refresh token cookie max age reduced to 400 days.
- **Custom version information on settings page**: Customers and partners can replace version info with their own.
- **Updated Auditor Service**: RBAC and Bulk Import plugins use Backstage's Auditor service. Audit log format updated, events grouped by `eventId`.
- **"Create" renamed to "Self-service"**: Across sidebar, global header, catalog page, and scaffolder page.
- **Enhanced plugin visibility in Extensions catalog**: Default plugin configurations displayed in the UI. Administrators can view config options before enabling.
- **Automatic baseUrl configuration for Operator on OpenShift**: Operator computes default URL from cluster ingress domain. Populates `app.baseUrl`, `backend.baseUrl`, `backend.cors.origin` automatically.
- **Sidebar item visibility configuration**: New `enabled` key to hide sidebar items. Also toggle core elements (logo, search, settings, administration).
- **Community plugins updated to Backstage 1.36**
- **New RBAC conditional rule IS_OWNER**: Ownership-based filtering of roles, permission policies, and conditional policies. Resource type removed from `policy.entity.create` permission.
- **HA support for Microsoft Azure Kubernetes Service (AKS)**: Scale beyond single replica.
- **GitHub scaffolder module** (`@backstage/plugin-scaffolder-backend-module-github`): Create repos, open PRs, trigger GitHub Actions from templates.
- **Default OIDC sign-in resolver updated**: Set to `oidcSubClaimMatchingKeycloakUserId`.
- **Kubernetes scaffolder actions plugin**: `@backstage-community/plugin-scaffolder-backend-module-kubernetes` with `create-namespace` action. Disabled by default.
- **readOnlyRootFilesystem enabled by default**: For both initContainer and backend container.

## Breaking changes

- **Topology-specific permission `topology.view.read` removed**: Use Kubernetes plugin permissions (`kubernetes.clusters.read`, `kubernetes.resources.read`, `kubernetes.proxy`) instead. Update CSV permission files.
- **Tekton-specific permission `tekton.view.read` removed**: Same — use Kubernetes plugin permissions instead. Update CSV permission files.
- **Migration to core Auditor service**: Audit log format, field names, and event IDs changed. Filtering queries based on old format may fail. Events now follow Backstage conventions (lowercase, kebab-case). Legacy event names (`ScaffolderTaskCreation`, `CatalogEntityDeletion`) no longer used.

## Deprecated features

- **Dynamic imports with `import(...)` deprecated**: Use `require(...) as typeof import(...)` instead. Backstage CLI now supports native ESM in Node.js.

## Developer Preview features

- **Red Hat Developer Hub Local (RHDH Local)**: Lightweight self-contained RHDH for local development without Kubernetes cluster. Work on templates, try plugins, validate software catalogs.
