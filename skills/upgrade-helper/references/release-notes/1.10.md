# RHDH 1.10 Release Notes

Red Hat Developer Hub 1.10 is a fully supported, enterprise-grade productized version of upstream Backstage 1.49.4.

Source: <https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.10/html/red_hat_developer_hub_release_notes/index>

## New features and enhancements

- **Browser tab displays Scaffolder's template title**: The Scaffolder dynamically displays the title of the software template in the browser tab.
- **Bulk Import preserves selected approval tool across page refreshes**: The selection (GitHub or GitLab) is stored in a URL parameter.
- **PingFederate available as an authentication provider**: Sign in to RHDH using PingFederate credentials. Requires LDAP catalog provider plugin.
- **Personalized homepages for RHDH user groups**: Create persona-specific homepages tailored for different user groups. Attach homepages to specific Backstage or RBAC groups.
- **Trigger workflows through cloud events**: Dynamically manage tasks by emitting cloud events to Kafka instances.
- **Default RBAC role and baseline permissions configuration**: Configure `permission.rbac.defaultPermissions` in app-config.yaml for all authenticated users.
- **Human-readable entity titles in Adoption Insights**: Displays titles instead of technical names in "Top 3" widgets.
- **RHDH Dynamic Plugin Factory enhancement**: Building multiple workspaces, auto-generated plugins-list.yaml, `--clean` argument, Git repository references.
- **Red Hat Developer Lightspeed for RHDH available as a default plugin**: Installed by default with Operator or Helm chart. Remains unavailable until LLM is configured. Customers can disable via Operator flavour or Helm chart toggle.
- **MCP servers available from Developer Lightspeed chat interface**: Manage MCP servers from chat. Tokens encrypted at rest using AES-256-GCM. Permissions: `lightspeed.mcp.read`, `lightspeed.mcp.manage`.
- **Additional Scaffolder capabilities available as MCP tools**: RHDH exposes Scaffolder capabilities as MCP tools through backstage scaffolder and scaffolder-mcp-extras plugins.
- **File-level compliance checks for Scorecard**: Check presence of specific files (LICENSE, CODEOWNERS, Dockerfile, .gitignore) across repositories.
- **Bulk Import repository list scoped to the signed-in user**: Listings fetched via SCM OAuth. Hides already-imported repositories. **Important:** Administrators must configure GitHub and GitLab OAuth for repository listing.
- **Spanish localization for RHDH**
- **German localization for RHDH**
- **Customized aggregated metric cards for Scorecard**: Average aggregation type for overall health assessment.
- **Single database deployments with pluginDivisionMode**: Schema-based plugin isolation within a single shared database. Useful for users without CREATEDB privileges. **Important:** `pluginDivisionMode: schema` requires CREATE SCHEMA privileges.
- **Alpha Backstage frontend system available (app-next)**: Enable with `APP_CONFIG_app_packageName=app-next` and `ENABLE_STANDARD_MODULE_FEDERATION=true`.
- **New Frontend System (NFS) support**: Added to Red Hat-owned GA frontend plugins (Adoption Insights, Bulk Import, Extensions, Scorecard, RBAC, Orchestrator, Global Header, etc.).
- **Node.js 24 update**: RHDH updated to Node.js 24.
- **Core RHDH UI features migrated to NFS**: Homepage, Analytics Provider, Theme, Translation plugins load as dynamic plugins.
- **Scorecard navigation and data freshness for aggregated KPIs**: Cards show last-updated timestamps and link to contributing entities.
- **Flavor-based configuration for the Operator (v1alpha5 API)**: Deploy pre-configured instances using `spec.flavours[]`. Support for multiple ConfigMaps in a single YAML file. `rhdh.redhat.com/sub-path` annotation for volume mount control.
- **Orchestrator backend module Loki (GA)**: `@red-hat-developer-hub/backstage-plugin-orchestrator-backend-module-loki` is now GA.
- **OpenShift Container Platform 4.21 supported** (Kubernetes 1.34).
- **Retry configuration for Orchestrator widgets**: Configure retries with count, delay, backoff multiplier, and status codes.
- **Custom review page for Orchestrator**: Implement `getReviewComponent()` on OrchestratorFormApi for custom review steps.

## Technology Preview features

- **Enhanced security compliance with Dependabot and OpenSSF integrations for Scorecard**: Simplifies complex security checks into visual summaries.
- **RHDH must-gather (Technology Preview)**: Streamlines collection of diagnostic data from RHDH deployments for support case troubleshooting.

## Developer Preview features

- **Personal AI Notebooks**: Create isolated workspaces for RAG-based chat with uploaded documents (PDFs, Markdown, .docx).
- **Backstage MCP tools**: Support for Backstage MCP tools with Red Hat-specific `rhdh-mcp-<domain>-extras` overlay plugins.
- **Customizable Scorecard severity levels, colors, and icons**: Custom severity thresholds, color mappings, and icon assignments.
- **Developer Lightspeed evaluation framework and data**: AI-generated Q&A datasets and evaluation result reports for model selection.

## Deprecated features

- **Global Floating Action Button disabled by default**: Use Global Header instead. Existing deployments that explicitly enable it are not affected.
- **Deprecating community-supported authentication providers**: The following will be removed as static modules in a future release and require adding the corresponding dynamic plugin:
  - Atlassian, Auth0, Azure-easyauth, Bitbucket, Bitbucket Server, Cloudflare Access Provider, Google, Google IAP, OAuth 2 Custom Proxy, OneLogin, Okta
- **Change of support level**: `backstage-community-plugin-acr` moved from Technology Preview to Community support. Use `oci://` reference going forward.

## Removed features

- **Operator no longer deletes user-created resources if features are disabled in CR**: Must delete manually.
- **OCM plugin removed**: OCM plugin and wrapper have been removed.
- **ArgoCD scaffolder moved to Community support and removed as embedded wrapper**: `roadie-backstage-plugins/plugins/scaffolder-actions/scaffolder-backend-argocd` — must use `oci://` path reference.
- **GA plugins downgraded to Community and removed as embedded wrappers**:
  - `backstage-community-plugin-quay`
  - `backstage-community-plugin-scaffolder-backend-module-quay-dynamic`
  - `backstage-community-plugin-tekton`
  - `roadiehq-scaffolder-backend-argocd-dynamic`
  - Must use `oci://` path reference to continue using these plugins.

## Known issues

- **OCI images from registry.access.redhat.com fail to auto-detect plugin paths** (RHDHBUGS-3307): Multi-arch OCI image indexes cause skopeo to miss `io.backstage.dynamic-packages` annotation. Workaround: use digest-based reference or append `!<plugin-path>` suffix.
- **Bulk Import marks repositories as 'already imported' with open PRs** (RHDHBUGS-3001): No workaround.

## Fixed issues (upgrade-relevant)

- **Deprecated `backend.auth.keys` removed** (RHDHBUGS-2686): Replaced by static token.
- **Deprecated 'janus-idp' templates replaced with 'rhdh'** (RHDHBUGS-2557): Helm template references updated.
- **In-place Helm upgrade failing when Orchestrator enabled** (RHDHBUGS-3439): Fixed immutable job error on 1.9→1.10 upgrade.
- **Operator reconciliation loop causes unnecessary pod restarts** (RHDHBUGS-3084): Fixed non-deterministic ordering. **Note:** When defining several app-config files inside one ConfigMap, keys are now sorted alphabetically. Use separate single-entry ConfigMaps for explicit merge order.
- **Operator crashes with OutOfMemory errors** (RHDHBUGS-3414): Fixed bloated informer cache.
- **Helm deployments on EKS and AKS fail with container permission errors** (RHDHBUGS-3371): Fixed lightspeed-core sidecar permissions for non-OCP environments.
- **Keycloak nested subgroups now appear in catalog** (RHDHBUGS-2309): Fixed for Keycloak 26.4.0+.
- **Disabling default dynamic plugins includes** (RHDHBUGS-2603): Send empty `includes: []` to exclude defaults.
- **Transparent mirroring with custom registries.conf** (RHDHBUGS-2649): No longer need to rebuild catalog index image after mirroring.
