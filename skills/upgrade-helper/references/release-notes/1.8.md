# RHDH 1.8 Release Notes

Red Hat Developer Hub 1.8 is a fully supported, enterprise-grade productized version of upstream Backstage 1.42.5.

Source: https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.8/html/red_hat_developer_hub_release_notes/index

## New features and enhancements

- **Built-in monitoring for Red Hat Developer Hub Operator**: Enable `spec.monitoring.enabled: true` in Backstage CR for automatic Prometheus metrics collection on `/metrics` endpoint.
- **Enhanced package management in the Extensions page**: Installed Plugins tab renamed to Installed Packages with row actions to edit, enable, and disable packages directly.
- **Localization support and French localization**: Core plugins (Adoption Insights, AI Integrations, Bulk Import, Extensions, Lightspeed, Orchestrator, QuickStart, RBAC, ScoreCard, Topology, Global Header, Homepage, Tekton, ArgoCD) support localization. French language available.
- **Localization support for config file strings**: Strings in `app-config.yaml` and `dynamic-plugins.default.yaml` can be localized (Entity Tabs, Global Header, QuickStart, Sidebar Menu Items, FAB labels).
- **Plugins localization support**: Backstage localization framework integrated. Users can load translations from plugins or external JSON files.
- **Enhanced Bulk Import with Scaffolder Templates**: Import repositories using scaffolder templates and Orchestrator workflows.
- **Software Template version update notifications**: Notification alerts when Software Templates are updated.
- **Software Template provenance and dependency tracking**: Dedicated dependency view for component traceability and lifecycle management.
- **Users can customize their homepage**: Move, resize, remove, and add cards. Reset to default configuration.
- **Community plugins updated to Backstage 1.42**
- **Quick Start experience for first-time developers**: Guided next steps for Bulk Import, Software Catalog, Self-service templates, Learning paths. RBAC-integrated.
- **Transparent plugin support indicators**: Verified badge removed. Tiered support system (GA, TP, DP) with Release Status, Supported By, Backstage Version Supported, Author, Tags, Category fields.
- **Support for high availability in Google Kubernetes Engine (GKE)**: Scale beyond single replica.
- **Customizable container deployment in RHDH pods**: Inject files and env vars into any container (sidecars, system containers), not just backstage-backend.
- **Unauthenticated access to RHDH images**: Use `registry.access.redhat.com` instead of `registry.redhat.io`.

## Technology Preview features

- **Bulk import GitLab projects**: Bulk import entities from GitLab.

## Developer Preview features

- **Dynamic Plugin Factory**: Automate conversion and packaging of standard Backstage plugins into dynamic plugins.
- **Events Module**: Use with scheduled updates for GitHub user/catalog entity updates on external changes.
- **Built-in TechDocs for RHDH Local**: Getting Started and How-To documentation embedded as TechDocs.
- **Tech Radar and Quay plugins in RHDH Local**: Out-of-the-box experience with Tech Radar and Quay plugins.
- **RHDH Local default homepage enhancements**: Floating action button with quick links.
- **Pre-loaded templates in RHDH Local**: Create custom plugins, add TechDocs or software components.
- **Scorecard plugin**: Configurable KPIs from GitHub PRs and Jira issues. Custom metric providers supported.
- **OpenShift AI Connector for RHDH**: Surface AI Models and Model Servers from Red Hat OpenShift AI into the Catalog.
- **Model Context Protocol (MCP) tools**: Seamless communication with AI clients.
- **Developer Lightspeed migrated to Lightspeed Core (LCORE)**: Migration from Road-Core Service to LCORE.

## Breaking changes

- **Argo CD, Tekton, and Topology plugins require Kubernetes Frontend and Backend plugins**: Before this update, only Kubernetes Backend was needed. Now both Frontend and Backend must be enabled. The `@janus-idp/shared-react` package was removed — its functionality moved to Kubernetes Frontend.
- **Azure Repositories module removed**: `@parfuemerie-douglas/scaffolder-backend-module-azure-repositories` is incompatible with Backstage 1.42.5 and unmaintained. Update to RHDH 1.9 for the replacement `@backstage-community/plugin-scaffolder-backend-module-azure-devops`.
- **Existing 1.8.5 Operator users with Orchestrator require manual ConfigMap update**: Set Orchestrator plugin versions to 1.8.12 in dynamic-plugins ConfigMap after Operator upgrade to 1.8.6.
- **PagerDuty plugins removed**: `@pagerduty/backstage-plugin` and `@pagerduty/backstage-plugin-backend` incompatible with Backstage 1.42.5 and unmaintained. Update to RHDH 1.9 for latest PagerDuty modules.

## Deprecated features

- **Backstage CR versions v1alpha1 and v1alpha2**: Deprecated in 1.7, will be removed in 1.9.
- **Deprecation of Bundled Plugin Wrappers**: Method of including wrapped dynamic plugins in the RHDH container image deprecated in 1.7. Transitioning to independent OCI artifacts. **No breaking changes in 1.8** — all bundled plugins continue to work. In 1.9, community-supported plugins will be removed as bundled wrappers and published exclusively to `ghcr.io`. Full migration table provided in release notes.
- **Deprecation of OCM Plugins**: Open Cluster Management (OCM) plugins deprecated as of 1.8, will be removed in a future release.

## Known issues

- **Existing Operator users with Orchestrator 1.7 require manual ConfigMap update** (RHDHBUGS-2240): Set Orchestrator plugin versions to 1.8.2 after Operator upgrade.
- **Deployment update error with dynamic plugin configuration** (RHDHBUGS-2227): Using `{{firstName}}` placeholder in `dynamicRoutes` config causes fatal "function firstName not defined" error.
- **RHDH Local sidebar highlighting issue** (RHDHBUGS-2132): Clicking RHDH Local Guide also highlights Catalog item.
- **Installation disabled scenario in installed packages page** (RHDHBUGS-2126): Actions shown when installation disabled; API failure handled gracefully.
- **Operator default configuration changes don't persist across upgrades** (RHDHBUGS-2102): No workaround.
- **Quay and Argo CD require backend plugins for permissions UI** (RHDHBUGS-2038): Must enable both frontend and backend plugins.
- **MUI v5 components broken styles in non-bundled plugins** (RHDHBUGS-986): Workaround: add `ClassNameGenerator.configure` to plugin entry file.
- **Out-Of-Memory errors in RHDH Operator** (RHDHBUGS-664): Set memory request to 1GiB, limit to 2GiB.
- **Orchestrator 1.8.7 upgrade requires ConfigMap update** (RHDHBUGS-2861): Set Orchestrator plugin versions to 1.8.12.

## Fixed issues (upgrade-relevant)

- **SCM integration for self-hosted enterprise providers fixed** (RHDHBUGS-1028, RHDHBUGS-2249)
- **Catalog Graph rendering resolved** (RHDHBUGS-2540)
- **Helm chart and Operator updated for 1.8.12 Orchestrator versions** (RHDHBUGS-2825, RHDHBUGS-2995)
