# RHDH 1.5 Release Notes

Red Hat Developer Hub 1.5 is a fully supported, enterprise-grade productized version of upstream Backstage 1.35.0.

Source: <https://docs.redhat.com/en/documentation/red_hat_developer_hub/1.5/html/red_hat_developer_hub_release_notes/index>

## New features and enhancements

- **TechDocs add-ons configuration** (`techdocsAddons`): Dynamic plugins can export components as TechDocs add-ons with props configuration.
- **Configurable global header plugin**: Enabled by default with search, create button, support/notification icons, profile dropdown. Extendable via dynamic plugins with new mount points.
- **Floating action button support**: Add plugins or external links as floating buttons for quick access.
- **HA support for OpenShift Container Platform**: Scale beyond single replica.
- **Top Visited and Recently Visited home page cards**: Track visited pages per user within browser.
- **Improved GitHub rate limit handling**: GitHub org entity provider manages primary and secondary rate limits with automatic pausing.
- **`catalog.disableRelationsCompatibility` config option**: Improves catalog performance and reduces memory.
- **`scaffolder.template.management` permission**: Restrict front-end template management.
- **Multi-replica with persistent volume for dynamic plugins**: Write conflict mitigation.
- **Configurable PVC mounting for containers**: Mount PVC to any container, not just Backstage.
- **Improved Backstage CR status conditions**: `DeployInProgress` reason added under `Deployed` condition.
- **Enhanced secret configuration**: Multiple secrets in single YAML, annotations for mount path and target containers.
- **Enhanced home page title customization**: Dynamic display of first name or display name.
- **Automatic platform detection for Operator**: Detects EKS, AKS, GKE and applies `fsGroup` patches.
- **Enhanced air-gapped installation script**: Supports all platforms, partially and fully disconnected environments.
- **Enhanced RBAC role creation**: Bulk selection for users, groups, plugins, and permissions. Intuitive CRUD action names.
- **Kubernetes scaffolder actions plugin**: `@backstage-community/plugin-scaffolder-backend-module-kubernetes` with `create-namespace`. Disabled by default.

## Breaking changes

- **Sign-in resolver config location changed**: `dangerouslyAllowSignInWithoutUserInCatalog` moved from root to resolver level in app-config.yaml.
- **GitLab `scopes` parameter now required**: For `gitlab:projectDeployToken:create` action. Must provide valid value like `['read_repository']`.
- **`dynamicPlugins.cache.volumeClaimSpec` removed from Helm chart**: Dynamic plugin storage is ephemeral by default. Must manually configure PVC via `extraVolumes`/`extraVolumeMounts` for persistence.

## Deprecated features

- **Podman and ImageStreams for Helm chart mirroring deprecated**: Transition to `oc-mirror`.
- **`useRedisSets` option removed from Redis backend cache**: Remove from `backend.cache` in app-config.yaml to avoid warnings.
- **ArgoCD front-end plugin from RoadieHQ deprecated**: Transition to ArgoCD plugin from Red Hat.
- **Legacy dynamic plugin configs and export options deprecated**: `apiFactories` config, `--embed-as-code`, `--no-embed-as-dependencies` deprecated.
- **Red Hat Build of Keycloak v24 deprecated**: Migrate to RHBK v26.
- **Red Hat Single-Sign On 7.6 removed**: Deprecated in 1.4, removed in 1.5.

## Technology Preview features

- **Extensions for Red Hat and community plugins**: Browse available plugins with filtering. Installation from front-end planned.
- **Loading authentication providers from dynamic plugins**: `ENABLE_AUTH_PROVIDER_MODULE_OVERRIDE` env var, custom `signInPage` component, `providerSettings` configuration.

## Developer Preview features

- **Adoption Insights**: Key metrics and trends for RHDH usage. Cannot be used alongside `plugin-analytics-provider-segment` (disable segment first).

## Known issues

- **Add button misaligned on Bulk Import page**
- **Topology plugin permission not displayed in RBAC UI**

## Fixed issues (upgrade-relevant)

- **Dynamic plugins PVC now ephemeral by default** (RHIDP-5573): Prevents upgrade and multi-replica issues.
- **Multi-replica support across cluster nodes** (RHIDP-5342): Ephemeral volume resolves PVC conflicts.
- **MODULE_NOT_FOUND for GitLab org catalog and Notification backend** (RHIDP-5308, RHIDP-5476): Missing dependencies embedded.
- **OCI artifact handling improved** (RHIDP-5609): `pullPolicy` config (Always/IfNotPresent), prevents unintended deletion.
- **RBAC startup performance improved** (RHIDP-6180): Bulk addition of roles and policies.
- **RBAC performance for large user/group counts** (RHIDP-4697)
- **Keycloak access token lifespan** (RHIDP-5023): Increase token lifespan to prevent frequent refresh.

## Fixed security issues

No specific advisories listed in the release notes for 1.5.
